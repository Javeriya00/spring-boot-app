Project Description

This page provides a detailed overview of the CI/CD pipeline implemented for the project.

The CI/CD process in the current application implementation includes a series of automated steps to streamline the development and deployment pipeline. It starts with the source code triggering Jenkins via a webhook, which initiates the build process. Jenkins then runs a Maven build, followed by SonarQube performing a thorough code analysis. Once the analysis is complete, tests are executed to verify the functionality of the code.

If the build and tests are successful, a new Docker image is pushed to DockerHub. The image updater then deploys the updated image to the Manifests Repository, and ArgoCD handles the deployment to the Kubernetes cluster. In the event of a failure, the pipeline generates detailed reports and sends notifications via Slack or email to inform the team of the issue. This process highlights the automation of key stages, including building, testing, code analysis, containerization, and deployment, while ensuring proper error handling and communication at every stage.
